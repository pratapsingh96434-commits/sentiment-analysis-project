# ML Prediction API 🌸

A production-style REST API built with **FastAPI** and **scikit-learn** that classifies Iris flower species from measurements.

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Train and save the model
python train_model.py

# 3. Start the API server
uvicorn app.main:app --reload

# 4. Open API docs in your browser
# → http://localhost:8000/docs
```

## API Endpoints

| Method | Endpoint            | Description              |
|--------|---------------------|--------------------------|
| GET    | `/`                 | Welcome / links          |
| GET    | `/api/v1/health`    | Health check             |
| POST   | `/api/v1/predict`   | Predict iris species     |

## Example Request

```bash
curl -X POST http://localhost:8000/api/v1/predict \
  -H "Content-Type: application/json" \
  -d '{"sepal_length": 5.1, "sepal_width": 3.5, "petal_length": 1.4, "petal_width": 0.2}'
```

## Example Response

```json
{
  "prediction": 0,
  "species": "setosa",
  "confidence": 0.98,
  "status": "success"
}
```

## Project Structure

```
ml_prediction_api/
├── app/
│   ├── __init__.py     # Makes app a Python package
│   ├── main.py         # App entry point, startup, middleware
│   ├── routes.py       # API endpoint handlers
│   ├── model.py        # ML model loading & prediction logic
│   └── schema.py       # Pydantic request/response models
├── model/
│   └── model.pkl       # Trained model (generated by train_model.py)
├── train_model.py      # One-time script to train & save model
├── requirements.txt
└── README.md
```

## AWS Deployment (Later)

1. **Dockerize**: Add a `Dockerfile` with `uvicorn app.main:app --host 0.0.0.0`
2. **Push to ECR**: Push your Docker image to Amazon ECR
3. **Deploy to ECS/EKS**: Run the container on AWS ECS (serverless) or EKS (Kubernetes)
4. **API Gateway**: Optionally put AWS API Gateway in front for rate limiting & auth
5. **Lambda**: For low traffic, deploy using AWS Lambda + Mangum adapter
